別端末でのセットアップ手順

Obsidian + Cursor を同期して、別のPC・Macでも同じように使う

📌 クイックスタート

端末やること
WindowsOneDriveログイン → Obsidian・Cursorインストール → MyVaultフォルダを開く
Mac同上。パスは ~/Library/CloudStorage/OneDrive-.../Documents/MyVault

VaultはすでにOneDrive内にあるので、同じMicrosoftアカウントでログインすれば自動同期されます。

📊 何が同期されるか

項目同期備考
ノート・フォルダ✅00_Inbox、20_Context、30_Projects など
Obsidian設定✅.obsidian（プラグイン・テーマ）
Cursorルール✅.cursorrules
Cursorエディタ設定❌フォント・テーマ等は端末ごと。手動で合わせる
🪟 Windows

Windows でのセットアップ

1
OneDrive OneDriveをインストール・起動し、同じMicrosoftアカウントでサインイン。「ドキュメント」フォルダの同期がオンか確認。
2
Obsidian obsidian.md からダウンロード・インストール。起動 →「既存のフォルダを開く」→ パスを指定：
C:\Users\<ユーザー名>\OneDrive\ドキュメント\Documents\MyVault
3
Cursor cursor.com からダウンロード・インストール。起動 →「フォルダを開く」→ 上記と同じ MyVault フォルダを指定。
→ .cursorrules が読み込まれ、同じAIルールで動作します。
🍎 Mac

Mac でのセットアップ

1
OneDrive OneDriveをインストール・起動し、同じMicrosoftアカウントでサインイン。同期先を確認。
2
Obsidian obsidian.md からダウンロード・インストール。起動 →「既存のフォルダを開く」→ パス例：
~/Library/CloudStorage/OneDrive-個人/Documents/MyVault
※「OneDrive-個人」はアカウント名で変わる場合があります
3
Cursor cursor.com からダウンロード・インストール。起動 →「フォルダを開く」→ 上記の MyVault フォルダを指定。
⚠️ Gitで同期する場合（OneDriveが使えない環境）
初回：git clone https://github.com/<ユーザー名>/MyVault.git
作業開始：git pull → 作業後：git add . → git commit → git push
※OneDriveとGitを両方使うと競合する場合あり。どちらか一方をメインに。

✅ セットアップ完了チェック

  • Vaultフォルダが同期されている
  • ObsidianでMyVaultを開ける
  • プラグイン・テーマが反映されている
  • CursorでMyVaultを開ける
  • チャットでルールが効いている

🔧 トラブルシューティング

OneDriveで同期されない
設定で「ドキュメント」がオンか確認。右クリック→「常にこのデバイスで保持」でオフライン対応。
Macでパスが分からない
FinderでOneDrive → ドキュメント → MyVault を開く → 右クリック →「パスをコピー」
Cursorでルールが効かない
MyVault フォルダ自体 を開いているか確認（中のファイルではなく）
競合ファイルができる
同時編集を避ける。片方で編集 → 保存 → もう片方で開く。
