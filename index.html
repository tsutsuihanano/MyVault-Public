別端末でのセットアップ手順

Obsidian + Cursor を別端末でも使えるようにする

⏱️ 所要時間：約10分

📌 前提

  • メイン端末で MyVault（Obsidian + Cursor + Git）が構築済み
  • 別端末にも MyVault フォルダは既にある
  • Git の設定（.git フォルダ）も既にある → PAT の再設定は不要
✅ やることは ツールをインストール → 既存の MyVault フォルダを開く → git の動作確認 だけ
STEP 1

🛠️ ツールのインストール

1
Git git-scm.com → ダウンロード・インストール。
インストーラーはデフォルト設定でOK
2
Obsidian obsidian.md → 「Get Obsidian for free」からダウンロード・インストール。
3
Cursor cursor.com → 「Download」からダウンロード・インストール。
初回起動時に「Sign in」→ Googleアカウントでログイン（メイン端末と同じアカウント推奨）
STEP 2

📂 Obsidian で MyVault を開く

1
Obsidian を起動 「Open folder as vault」（既存のフォルダを開く）を選択。
2
既存の MyVault フォルダを選択
場所がわからない場合は、エクスプローラー / Finder で「MyVault」を検索
3
プラグインの有効化 「Trust author and enable plugins」を選択。
→ フォルダ構造（00_Inbox、20_Context...）とプラグイン・テーマが反映されればOK
STEP 3

🤖 Cursor で MyVault を開く

1
Cursor を起動してログイン 「Sign in」→ Googleアカウントでログイン。
メイン端末と同じアカウントを使うとProプランが共有されます
2
「File」→「Open Folder」→ MyVault フォルダを選択
3
AI ルールの動作確認 チャットを開いて（Ctrl + L / Cmd + L）以下を送信：
テスト：.cursorrules が読み込めているか確認して
→ AI がVault構造やルールに言及すれば正常
⚠️ ファイル単体ではなくフォルダを開く
.cursorrules はフォルダをルートとして開いたときに読み込まれます。
STEP 4

🐙 Git の動作確認

.git フォルダは既にあるので、正しく動くか確認するだけです。

1
Cursor のターミナルを開く Ctrl + `（バッククォート）でターミナルパネルを表示。
2
Git の状態を確認
$ git status
# 「On branch main」と表示されればOK
3
リモートとの接続確認
$ git pull
# 「Already up to date.」 or 変更の取得が行われればOK
⚠️ git status でエラーが出る場合
Git がインストールされているか確認 → git --version で確認できます。
毎日やること

📅 日常の Git ルーティン

どの端末で作業しても、同じ GitHub を更新します。

作業開始
$ git pull
作業終了
$ git add .
$ git commit -m "更新メモ"
$ git push
💡 Cursor のターミナル（Ctrl + `）から直接実行できます。

✅ セットアップ完了チェック

  • Git・Obsidian・Cursor がインストールされている
  • Obsidian で MyVault を開いてフォルダ構造が見える
  • Obsidian のプラグイン・テーマがメイン端末と同じ
  • Cursor で MyVault フォルダを開いている
  • Cursor チャットで AI が .cursorrules を認識している
  • git pull が正常に動作する

🔧 トラブルシューティング

git status で「not a git repository」と出る
.git フォルダが見つからない状態。MyVault フォルダの直下にいるか確認 （cd でフォルダに移動してから再度実行）。
git pull で認証エラーが出る
PAT（Personal Access Token）の期限切れの可能性。
GitHub → Settings → Tokens で確認・再発行し、 リモートURLを更新：
git remote set-url origin https://ユーザー名:新PAT@github.com/ユーザー名/MyVault.git
Obsidian でプラグインが反映されない
「Trust author and enable plugins」を選択したか確認。していない場合は Obsidian を再起動。
Cursor で AI ルールが効かない
MyVault フォルダ自体を「Open Folder」で開いているか確認。ファイル単体ではルールは読み込まれません。
git push で rejected と出る
別端末から先に push されている場合に発生。
git pull --rebase → 競合があれば修正 → git push
