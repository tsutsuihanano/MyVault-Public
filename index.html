別端末でのセットアップ手順

Obsidian + Cursor + GitHub を同期して、どの端末でも同じ「第二の脳」を使う

⏱️ 所要時間：約15〜20分

📌 このガイドの前提

メイン端末で以下が完了している状態で、別の端末に同じ環境を構築する手順です。

項目メイン端末の状態
Obsidian VaultMyVault を構築済み
Cursorインストール済み・.cursorrules 設定済み
GitHubプライベートリポジトリ MyVault に push 済み

🔄 同期の仕組み

GitHub を経由して、どの端末でも同じVaultを使えるようにします。

🏠
メイン端末
→ push →
🐙
GitHub
→ pull →
💻
別端末

📊 何が同期されるか

項目同期備考
ノート・フォルダ✅00_Inbox、20_Context、30_Projects など
Obsidian設定✅.obsidian/（プラグイン・テーマ）
Cursorルール✅.cursorrules
Cursorエディタ設定❌フォント・テーマ等は端末ごと
Cursorアカウント❌端末ごとにログインが必要
Git認証情報❌端末ごとに設定が必要
STEP 1

🛠️ ツールのインストール

3つのツールをインストールします。すべて無料です。

1
Git（バージョン管理） git-scm.com → ダウンロード・インストール。
インストーラーはすべてデフォルト設定でOK
2
Obsidian（ノートアプリ） obsidian.md → 「Get Obsidian for free」からダウンロード・インストール。
3
Cursor（AIエディタ） cursor.com → 「Download」からダウンロード・インストール。
初回起動時に「Sign in」→ Googleアカウントでログイン（メイン端末と同じアカウント推奨）
STEP 2

🐙 GitHubからVaultを取得

⚠️ Personal Access Token（PAT）が必要です
メイン端末セットアップ時に発行したPATを用意してください。
紛失した場合は こちらから再発行（repoにチェック、Expiration: No expiration）

🪟 Windows（PowerShell）

1
Git の初期設定
$ git config --global user.name "あなたのGitHubユーザー名"
$ git config --global user.email "あなたのメールアドレス"
2
リポジトリを clone
$ cd ~\Documents
$ git clone https://ユーザー名:PAT@github.com/ユーザー名/MyVault.git
→ Documents\MyVault フォルダが作成されます

🍎 Mac（ターミナル）

1
Git の初期設定
$ git config --global user.name "あなたのGitHubユーザー名"
$ git config --global user.email "あなたのメールアドレス"
2
リポジトリを clone
$ cd ~/Documents
$ git clone https://ユーザー名:PAT@github.com/ユーザー名/MyVault.git
💡 PAT をURLに含めたくない場合
git clone https://github.com/ユーザー名/MyVault.git で clone → 認証を聞かれたらユーザー名とPATを入力
STEP 3

📂 ObsidianでVaultを開く

1
Obsidianを起動 「Open folder as vault」（既存のフォルダを開く）を選択。
2
clone した MyVault フォルダを指定
Win: C:\Users\<ユーザー名>\Documents\MyVault
Mac: ~/Documents/MyVault
3
プラグインを有効化 「Trust author and enable plugins」を選択。
→ フォルダ構造（00_Inbox、20_Context...）とプラグイン・テーマが反映されればOK
STEP 4

🤖 CursorでVaultを開く

1
Cursorを起動してログイン 「Sign in」→ Googleアカウントでログイン。
メイン端末と同じアカウントを使うとPro/Businessプランが共有されます
2
MyVaultフォルダを開く 「File」→「Open Folder」→ clone した MyVault フォルダを選択。
3
AIルールの動作確認 チャットを開いて（Ctrl + L / Cmd + L）以下を送信：
テスト：.cursorrules が読み込めているか確認してください
→ AIがVault構造やルールに言及すれば正常動作
⚠️ ファイル単体ではなくフォルダを開く
.cursorrules はフォルダをルートとして開いたときに読み込まれます。
毎日やること

📅 日常の同期ルーティン

作業開始
$ git pull # 最新の変更を取得
作業終了
$ git add .
$ git commit -m "更新メモ"
$ git push
⚠️ pull を忘れて作業すると、メイン端末の変更と衝突（コンフリクト）する可能性があります。
作業開始時は必ず git pull を習慣にしてください。
💡 Cursor のターミナル（Ctrl + `）から直接 git コマンドを実行できます。
別途ターミナルを開く必要はありません。

✅ セットアップ完了チェック

  • Git・Obsidian・Cursor がインストールされている
  • git clone で MyVault を取得できた
  • Obsidian でフォルダ構造・プラグインが表示される
  • Cursor で MyVault フォルダを開いている
  • Cursor チャットで AI が .cursorrules を認識している
  • git pull / git push が動作する

🔧 トラブルシューティング

git clone で認証エラーが出る
① PATの有効期限が切れていないか確認（GitHub → Settings → Tokens）
② PATに repo 権限があるか確認
③ ユーザー名・PATに特殊文字がある場合はURLエンコードが必要
Obsidianでプラグインが反映されない
① 「Trust author and enable plugins」を選択したか確認
② .obsidian/ フォルダが clone に含まれているか確認
③ Obsidian を再起動
Cursor で AI ルールが効かない
① MyVault フォルダ自体を「Open Folder」で開いているか確認
② .cursorrules がフォルダ直下にあるか確認
③ Cursor を再起動
git push で rejected と出る
メイン端末の変更が先に push されている。
→ git pull --rebase → 競合があれば修正 → git push
