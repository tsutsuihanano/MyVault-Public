別端末でのセットアップ手順

Obsidian + Cursor + GitHub を同期して、どの端末でも同じ「第二の脳」を使う

⏱️ 所要時間：約20〜30分

📌 このガイドの前提

メイン端末で以下が完了している状態で、別の端末に同じ環境を構築する手順です。

項目メイン端末の状態
Obsidian VaultOneDrive 内の Documents/MyVault に構築済み
Cursorインストール済み・.cursorrules 設定済み
GitHubプライベートリポジトリ MyVault に連携済み
同期方法OneDrive（自動同期）＋ GitHub（バックアップ・版管理）

☁️ OneDrive 同期 推奨

  • 自動で常時同期
  • 操作不要（保存すれば反映）
  • 同じMSアカウントでログインするだけ
  • ⚠️ 同時編集すると競合する可能性あり

🐙 GitHub 同期

  • 手動で pull / push が必要
  • 変更履歴が残る（版管理）
  • OneDriveが使えない環境で有効
  • ⚠️ pull 忘れると古い状態で作業してしまう
💡 どちらを使う？
同じMicrosoftアカウントが使える端末 → OneDrive（何もしなくても同期）
共有PCやMSアカウントが異なる端末 → GitHub（手動 pull/push）
※ 両方併用する場合は、片方で編集中はもう片方を閉じること

📊 何が同期されるか

項目OneDriveGitHub備考
ノート・フォルダ✅ 自動✅ push時00_Inbox、20_Context、30_Projects など

STEP 1

🛠️ ツールのインストール

3つのツールをインストールします。すべて無料です。

1
Obsidian（ノートアプリ） obsidian.md → 「Get Obsidian for free」からダウンロード・インストール。
※ まだ起動しなくてOK。先にOneDriveの同期を待つ
2
Cursor（AIエディタ） cursor.com → 「Download」からダウンロード・インストール。
初回起動時に「Sign in」→ Googleアカウントでログイン（メイン端末と同じアカウント推奨）
3
Git（バージョン管理） git-scm.com → ダウンロード・インストール。
インストーラーはすべてデフォルト設定でOK
方法A：OneDrive で同期する場合

☁️ OneDrive でVaultを同期

同じMicrosoftアカウントでログインすれば自動でVaultが降ってきます。

🪟 Windows

1
OneDriveにサインイン タスクバーのOneDriveアイコンをクリック → メイン端末と同じMicrosoftアカウントでサインイン。
Windows 10/11 なら OneDrive はプリインストール済み
2
「ドキュメント」フォルダの同期を確認 OneDrive設定 →「バックアップ」→「フォルダーのバックアップを管理」→ 「ドキュメント」がオンであることを確認。
3
同期完了を待つ エクスプローラーで以下のパスに MyVault フォルダが表示されるまで待つ：
C:\Users\<ユーザー名>\OneDrive\ドキュメント\Documents\MyVault
※ OneDriveアイコンに✅が付いたら同期完了
4
ファイルをオフライン利用可能にする MyVaultフォルダを右クリック →「常にこのデバイスに保持する」を選択。
これでオフラインでも編集可能になります

🍎 Mac

1
OneDriveをインストール・サインイン App Store または 公式サイトからインストール → 同じMSアカウントでサインイン。
2
Vaultフォルダを確認 Finder で以下のパスを確認：
~/Library/CloudStorage/OneDrive-個人/Documents/MyVault
※「OneDrive-個人」部分はアカウント種別で異なる場合があります
⚠️ Macでパスがわからない場合
Finder → OneDrive → ドキュメント → MyVault を開く → フォルダを右クリック →「パス名をコピー」
方法B：GitHub で同期する場合

🐙 GitHub でVaultを同期

OneDriveが使えない環境や、バックアップ目的で使用します。

⚠️ GitHub Personal Access Token（PAT）が必要です
メイン端末のセットアップ時に発行したPATを用意してください。紛失した場合は こちらから再発行 （repoにチェック、Expiration: No expiration）

🪟 Windows（PowerShell）

1
Git の初期設定 PowerShellを開いて以下を実行：
$ git config --global user.name "あなたのGitHubユーザー名"
$ git config --global user.email "あなたのメールアドレス"
2
リポジトリを clone Vaultを置きたいフォルダに移動して clone：
$ cd ~\Documents
$ git clone https://あなたのユーザー名:あなたのPAT@github.com/あなたのユーザー名/MyVault.git
→ Documents/MyVault フォルダが作成されます

🍎 Mac（ターミナル）

1
Git の初期設定 ターミナルを開いて以下を実行：
$ git config --global user.name "あなたのGitHubユーザー名"
$ git config --global user.email "あなたのメールアドレス"
2
リポジトリを clone
$ cd ~/Documents
$ git clone https://あなたのユーザー名:あなたのPAT@github.com/あなたのユーザー名/MyVault.git
⚠️ PAT をURLに含めたくない場合
git clone https://github.com/ユーザー名/MyVault.git で clone → 認証を聞かれたらユーザー名とPATを入力
STEP 3

📂 ObsidianでVaultを開く

1
Obsidianを起動 「Open folder as vault」（既存のフォルダを開く）を選択
2
MyVaultフォルダを指定
同期方法パス
OneDrive（Win）C:\Users\<名前>\OneDrive\ドキュメント\Documents\MyVault
OneDrive（Mac）~/Library/CloudStorage/OneDrive-.../Documents/MyVault
GitHub clone~/Documents/MyVault
3
同期を確認 左サイドバーにフォルダ構造（00_Inbox、10_Goals、20_Context...）が表示されればOK。 プラグインやテーマもメイン端末と同じものが反映されているか確認。
✅ 「信頼しますか？」と聞かれたら
「Trust author and enable plugins」（信頼してプラグインを有効化）を選択してください。
STEP 4

🤖 CursorでVaultを開く

1
Cursorを起動してログイン 「Sign in」→ Google（またはメール）でログイン。
メイン端末と同じCursorアカウントを使うと、Pro/Businessプランが共有されます
2
MyVaultフォルダを開く 「File」→「Open Folder」→ STEP 3と同じMyVaultフォルダを選択。
3
AIルールの動作確認 チャットパネルを開いて（Ctrl + L / Cmd + L）、以下を送信：
テスト：.cursorrules が読み込めているか確認してください
→ AIが Vault構造やルールについて言及すれば正常に動作しています
⚠️ 重要：ファイル単体ではなくフォルダを開く
.cursorrules はフォルダをルートとして開いたときに読み込まれます。 ファイル単体を開いてもルールは適用されません。
方法Bの場合のみ

⚙️ Cursor から Git を使う設定

Cursor のターミナルから push/pull できるようにします。OneDrive のみの場合はスキップ可。

1
Cursor内ターミナルを開く Ctrl + `（バッククォート）でターミナルパネルを表示。
2
リモートの確認
$ git remote -v
# origin https://github.com/ユーザー名/MyVault.git (fetch)
# origin https://github.com/ユーザー名/MyVault.git (push)
3
動作テスト
$ git status
# On branch main / nothing to commit であれば正常

📅 日常の同期ルーティン

☁️ OneDrive の場合

✅ 何もする必要はありません。保存すれば自動で同期されます。
ただし、同時に複数端末で同じファイルを編集しないこと。

🐙 GitHub の場合

作業開始時
$ git pull # 最新の変更を取得
作業終了時
$ git add .
$ git commit -m "更新内容のメモ"
$ git push
⚠️ pull を忘れて作業すると、メイン端末の変更と衝突（コンフリクト）する可能性があります。
作業開始時は必ず git pull を習慣にしてください。

✅ セットアップ完了チェック

  • Obsidian・Cursor・Git がインストールされている
  • MyVault フォルダが同期されている（OneDrive or clone）
  • Obsidian で MyVault を開き、フォルダ構造が見える
  • Obsidian のプラグイン・テーマがメイン端末と同じ
  • Cursor で MyVault フォルダを開いている
  • Cursor にログインできている
  • Cursor チャットで AI が .cursorrules を認識している
  • （GitHub同期の場合）git pull / git push が動作する

🔧 トラブルシューティング

OneDriveでMyVaultが表示されない
① OneDrive設定 →「バックアップ」→「ドキュメント」がオンか確認
② 同期中の場合は時間を置く（初回は数分〜数十分かかる場合あり）
③ MyVaultフォルダを右クリック →「常にこのデバイスに保持する」を選択
git clone で認証エラーが出る
① PATの有効期限が切れていないか確認（GitHub → Settings → Tokens）
② PATに repo 権限があるか確認
③ ユーザー名・PATに特殊文字がある場合はURLエンコードが必要
Obsidianでプラグインが反映されない
① Vault を開いた直後に「Trust author and enable plugins」を選択したか確認
② .obsidian/ フォルダが同期されているか確認
③ Obsidian を再起動してみる
Cursor で AI ルールが効かない
① MyVaultフォルダ自体を「Open Folder」で開いているか確認（ファイル単体ではダメ）
② .cursorrules がフォルダ直下にあるか確認
③ Cursor を再起動
git push で rejected と出る
メイン端末の変更が先に push されている場合に発生。
git pull --rebase → 競合があればファイルを修正 → git push
OneDriveで競合ファイルができた
複数端末で同時編集すると「〇〇 (競合コピー).md」が生成される。
→ 中身を見比べて正しい方を残し、競合コピーを削除。
→ 同時編集を避けるルールを徹底する。
